# Store Sales — Time Series Forecasting

> 🔗 **Соревнование на Kaggle:** [store-sales-time-series-forecasting](https://www.kaggle.com/competitions/store-sales-time-series-forecasting/overview)

---

## 1. Описание Задачи (Problem Statement)

**Бизнес-проблема:** Corporación Favorita — крупная сеть продуктовых супермаркетов в Эквадоре — нуждается в точном прогнозировании объёмов продаж по тысячам товарных позиций в разрезе каждого магазина. Избыточные запасы приводят к порче скоропортящихся продуктов и замораживанию капитала; недостаточные — к потере выручки и неудовлетворённости покупателей.

**Почему это важно:**
- Субъективные методы планирования не масштабируются при добавлении новых магазинов и товаров.
- Сезонные колебания, промоакции, государственные праздники и макроэкономические факторы (например, цены на нефть) существенно влияют на спрос.
- Автоматизированный ML-прогноз способен снизить потери от перезапасов и повысить уровень сервиса.

**Тип задачи:** Многомерное регрессионное прогнозирование временных рядов (Multi-variate Time-Series Regression Forecasting).

**Горизонт прогноза:** 15 дней вперёд относительно последней даты в обучающей выборке.

## 2. Данные (Data)

### Файлы датасета

| Файл | Описание |
|---|---|
| `train.csv` | Обучающая выборка: временные ряды продаж по магазинам и товарным группам |
| `test.csv` | Тестовая выборка (те же признаки, без колонки `sales`) — нужно предсказать продажи |
| `sample_submission.csv` | Пример файла для сабмита в правильном формате |
| `stores.csv` | Метаданные о магазинах: город, штат, тип, кластер |
| `oil.csv` | Ежедневные цены на нефть (охватывают и тренировочный, и тестовый периоды) |
| `holidays_events.csv` | Праздники и события с метаданными (тип, локальность, признак переноса) |

### Описание признаков `train.csv` / `test.csv`

| Признак | Тип | Описание |
|---|---|---|
| `date` | date | Дата записи |
| `store_nbr` | int | Идентификатор магазина |
| `family` | str | Товарная категория (например, DAIRY, BEVERAGES) |
| `sales` | float | ⭐ **Целевая переменная** — суммарные продажи товарной категории в магазине за дату |
| `onpromotion` | int | Количество товаров из категории, участвующих в промоакции |

### Важные контекстные замечания

- **Праздники (holidays_events.csv):** Поле `transferred = True` означает, что праздник был официально перенесён. Реальный выходной — дата с типом `Transfer`. Дни типа `Bridge` — дополнительные выходные, которые «отрабатываются» в дни типа `Work Day`.
- **Нефть (oil.csv):** Экономика Эквадора сильно зависит от нефтяных доходов. Падение цен на нефть напрямую влияет на потребительский спрос.
- **Зарплаты:** В государственном секторе зарплату выдают 15-го и в последний день месяца — это создаёт всплески продаж в супермаркетах.
- **Землетрясение:** 16 апреля 2016 года в Эквадоре произошло землетрясение магнитудой 7.8. Это вызвало аномальный спрос на товары первой необходимости в супермаркетах на несколько недель.
- **Магазины (stores.csv):** Поле `cluster` группирует магазины со схожими характеристиками — полезно для сегментированного моделирования.

### Целевая переменная (Target)

```
sales — суммарные продажи товарной семьи (family) в конкретном магазине (store_nbr) за конкретную дату.
Может принимать дробные значения (например, 1.5 кг сыра).
```

## 3. Метрика Оценки (Evaluation Metric)

### Root Mean Squared Logarithmic Error (RMSLE)

$$\text{RMSLE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} \left(\log(1 + \hat{y}_i) - \log(1 + y_i)\right)^2}$$

**Что это означает на практике:**
- Метрика измеряет **относительную** ошибку прогноза, а не абсолютную.
- Штраф за ошибку **одинаково чувствителен** для малых и больших объёмов продаж.
- Логарифмирование делает метрику устойчивой к выбросам в целевой переменной.
- Логарифм `log(1 + y)` гарантирует корректность при нулевых продажах.
- **Меньше — лучше.** Идеальный результат: RMSLE = 0.

**Практический вывод:** Следует уделять особое внимание точности прогноза для категорий с малыми объёмами продаж — ошибки на них штрафуются не меньше, чем на крупных.

## 5. Быстрый Старт (Quick Start)

### Установка зависимостей

```bash
git clone https://github.com/glebtyryshkin/Store_Sales_TS.git
cd Store_Sales_TS

pip install -r requirements.txt
```

### Загрузка данных

Исходные CSV **не входят в репозиторий** (объём десятки–сотни МБ; лимит GitHub на один файл ~100 МБ). После клона скачайте данные с Kaggle в каталог `data/`:

```bash
kaggle competitions download -c store-sales-time-series-forecasting
unzip store-sales-time-series-forecasting.zip -d data/
```

Нужны как минимум: `train.csv`, `test.csv`, `stores.csv`, `oil.csv`, `holidays_events.csv`, `transactions.csv`, `sample_submission.csv`.

### Запуск EDA-ноутбука

```bash
jupyter lab notebooks/01_eda.ipynb
```

### Запуск FastAPI сервиса локально

```bash
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

### Streamlit-дашборд

Сначала поднимите API (см. выше), затем:

```bash
streamlit run app.py --server.port 8501
```

### Docker (опционально)

Готовых `Dockerfile` в репозитории нет; при желании соберите образ вручную на базе Python и `requirements.txt`.

## 6. Ключевые Идеи для Моделирования

- **Лаговые признаки (lags):** продажи за -7, -14, -28, -364 дней — главные предикторы.
- **Скользящие средние (rolling means):** окна 7, 14, 30 дней сглаживают шум.
- **Дата-признаки:** день недели, день месяца, номер недели, квартал, флаг выходного.
- **Праздники:** one-hot или ordinal кодирование типа праздника; флаг для переносных праздников.
- **Масштаб нефти:** скользящее среднее цены нефти как макроэкономический регрессор.
- **Промоакции:** `onpromotion` — сильный сигнал, особенно в сочетании с категорией товара.
- **Кластеры магазинов:** обучение отдельных моделей per-cluster улучшает качество.
- **Аномалия:** необходимо корректно обработать период землетрясения (апрель–май 2016).
